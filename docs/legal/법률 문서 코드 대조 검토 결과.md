# 출시 전 확인 및 백엔드 작업

- 최종 정리일: 2026년 7월 7일
- 범위: 운영자가 확인해야 할 사실과 백엔드 변경 작업만 관리한다.

## 백엔드 변경 필요

### 1. 계정 삭제 API

- [ ] 인증된 회원이 자신의 계정을 삭제하는 API 구현
- [ ] 계정과 소셜 로그인 연동 식별자 삭제
- [ ] 일정, 알림 설정, 즐겨찾기, 최근 조회, 병원 연락 이력 삭제
- [ ] FCM 토큰과 리프레시 토큰 폐기
- [ ] 거래·법적 보관 의무가 없는 회원 연결 데이터 전체 삭제
- [ ] 삭제 성공 후 재가입해도 이전 데이터가 복구되지 않는지 검증

### 2. 만 14세 이상 확인 기록

> 앱 UI의 필수 확인과 로그인 차단은 구현됨. 현재 클라이언트는 확인값을 백엔드로 전송하지 않음.

- [ ] 로그인·신규 가입 요청에 `ageConfirmed`, 약관 버전 추가
- [ ] 백엔드에서 신규 가입 시 확인값이 없는 요청 거부
- [ ] 회원별 확인 시각과 약관 버전 저장

### 3. 마케팅 수신 설정

> 현재 마케팅 수신 설정은 기기에만 저장되며 서버 발송 설정에 반영되지 않음.

- [ ] 출시 범위에서 마케팅 알림을 제공할지 운영자 결정
- [ ] 제공할 경우 회원별 동의 여부·시각을 저장하고 발송 시 검증하는 API 구현
- [ ] 제공하지 않을 경우 앱에서 마케팅 수신 설정 제거

## 운영자 확인 필요

### 1. AWS 국내 저장 및 FCM 국외 처리

- [x] API 서버는 AWS 서울 리전(`ap-northeast-2`) EC2에서 운영
- [x] 데이터베이스는 RDS를 사용하지 않고 서울 리전 EC2 내부에 저장
- [x] AWS의 HeHe 서비스 데이터는 국내에서 처리되는 것으로 문서에 반영
- [ ] FCM이 국외에서 처리하는 정확한 대상·국가·시점·방법·보유 기간 확인
- [ ] 확인한 FCM 국외 처리 내용을 개인정보처리방침에 최종 반영

### 2. 백엔드 저장·보유 정책

- [ ] 카카오·네이버에서 받은 정보의 실제 DB 저장 필드
- [ ] OAuth 액세스 토큰의 서버 저장 여부와 보유 기간
- [ ] IP, API 접속 로그, 오류 로그의 정확한 보유 기간
- [ ] 위치 좌표가 API 쿼리 로그에 남지 않는지 확인
- [ ] 법적 의무로 탈퇴 후에도 보관할 데이터가 있는지 확인

## 확인 후 최종 작업
